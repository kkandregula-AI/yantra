I

The Harness Loop

The harness is not intelligence — it is plumbing with discipline. It assembles context, calls the model, executes what the model asks, feeds the result back, and repeats until the work is done. Read the anatomy below; at the end of the page, the Lab Bench lets you run this circuit for real.

II

Skills

A skill is instructions loaded on demand — a SKILL.md file the harness folds into context when needed. Toggle one and watch the Context Inspector change. That is the whole trick: no fine-tuning, no magic. Text in, behavior out.

IN THE WILD: Anthropic's docx / pptx / pdf / xlsx skills (fetchable below) · a company's brand-guidelines skill · any SKILL.md you write yourself.

III

Tools

The distinction that matters most: a skill is knowledge — text loaded into context that changes how the model thinks and speaks. A tool is a capability — code the harness executes on the model's behalf. A skill never runs; a tool never persuades. When the model wants a tool, it asks; the harness acts; the result returns as an observation — that round-trip is the ACT → OBSERVE edge in the loop above.

IN THE WILD: bash · file editing · web search · code execution · MCP servers for Gmail, GitHub, Slack, databases · and this page's humble calculator.

V

The Rest of the Stack

YANTRA is the minimal agent — deliberately. Production harnesses add four more layers. Same philosophy, more discipline.

memory & state

This page keeps one messages array in RAM — gone on reload. Real harnesses persist state across sessions, summarise old turns to fit the context window, and let agents accumulate experience. The context window is finite; memory management is what makes long work possible.

progressive disclosure

YANTRA loads a skill's full text the moment you toggle it. Production harnesses load only each skill's name + description, and fetch the body when the task matches — that's why the Anthropic skills you fetch above get truncated here. Context is a budget; spend it late.

mcp · external tools

Our calculator is hardcoded into the harness. MCP (Model Context Protocol) is the provision for tools the way SKILL.md files are for skills: a standard interface so any harness can discover and call third-party capabilities — email, databases, browsers — without custom wiring.

permissions & verification

A calculator is harmless, so YANTRA auto-executes it. When tools can send emails or move money, the harness becomes a checkpoint: allow-lists, human confirmation for risky actions, and audit logs of every ACT. The loop stays the same — the gate between INFER and ACT grows teeth.
